# infrastructure/krx

KRX(한국거래소) Open API 어댑터 모듈.

베이스 URL: `https://data-dbg.krx.co.kr/svc/apis`  
인증: 요청 헤더 `AUTH_KEY: {키값}`

---

## 엔드포인트

| 용도 | KOSPI | KOSDAQ | KONEX |
|------|-------|--------|-------|
| 종목 목록 | `/sto/stk_isu_base_info` | `/sto/ksq_isu_base_info` | `/sto/knx_isu_base_info` |
| 일별 주가 | `/sto/stk_bydd_trd` | `/sto/ksq_bydd_trd` | `/sto/knx_bydd_trd` |

공통 파라미터: `basDd=yyyyMMdd` (기준일)

## API 응답 정책 (실측)

### 종목 조회

| 조회 날짜 | 응답 |
|----------|------|
| 과거 영업일 | 데이터 반환 |
| 과거 주말·공휴일 | **데이터 반환** (직전 영업일 종목 목록 그대로 반환) |
| 당일 (미확정) | 빈 배열 `{"OutBlock_1":[]}` |

### 주가 조회

| 조회 날짜 | 응답 |
|----------|------|
| 과거 영업일 | 데이터 반환 |
| 과거 주말·공휴일 | 빈 배열 |
| 당일 (미확정) | 빈 배열 |

### 요약

- 종목 조회는 날짜를 기준으로 "그 날 기준의 상장 종목 목록"을 반환하며, 비영업일에는 직전 영업일 목록을 그대로 돌려준다.
- 주가 조회는 실제 거래가 발생한 날만 데이터가 존재한다.
- **두 API 모두 당일 날짜는 빈 배열을 반환한다.**

---

## T+1 데이터 제공 정책

D일 거래 데이터는 **D+1일 약 08:00 KST 이후**에 제공된다.

```
D일 거래 마감 → D+1 08:00~ KRX 업로드 → 조회 가능
```

- 연휴 직후에는 연휴 전 마지막 영업일 데이터가 연휴 종료 다음 날 08:00에 제공된다.
- 예: 목요일이 마지막 영업일이고 금~화가 연휴이면, 수요일 08:00에 목요일 데이터가 올라온다.
- 빈 배열 응답만으로는 "휴장일"인지 "아직 미제공"인지 구분할 수 없다. → 코드에서는 **UNCONFIRMED(미확정)** 으로 처리하며, 커서를 전진시키지 않고 다음 실행에서 재시도한다.

---

## 빈 배열의 의미

빈 배열은 두 가지 상황 중 하나다:

1. **실제 휴장일** — 해당 날짜에 거래가 없음
2. **데이터 미제공** — 거래는 있었으나 KRX가 아직 업로드하지 않음

이 둘을 API 응답만으로 구분할 수 없기 때문에, 시스템은 빈 배열을 항상 UNCONFIRMED로 처리한다. 이후 데이터가 올라오면 자동으로 SUCCESS로 전환된다.
